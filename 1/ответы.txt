12. Поясните понятие in-memory Database.
База данных размещаемая в оперативной памяти.
13. Поясните понятие хэш-таблица.
 это специальная структура данных для хранения пар ключей и их значений. 
По сути это ассоциативный массив, в котором ключ представлен в виде хеш-функции. 
14. Каким образом обеспечивается персистентность  данных в СУБД Redis?
Персистентность для структуры данных — возможность хранить несколько версий сразу.
Персистентность данных обеспечивается двумя механизмами:
периодическим сбросом in-memory данных на диск — snapshoting;
последовательной упреждающей записью всех приходящих операций в файл — transaction journal.
15. Поясните назначение команд СУБД Redis:  set, get, del, getset.
SET ключ значение — Записывает строковое значение в переданный ключ. Если ключ до этого существовал, то он будет перезаписан.
GET ключ — Возвращает значение ключа.
Del ключ – удаление ключа.
GETSET ключ значение — Устанавливает в переданный ключ строковое значение и возвращает предыдущее.
16. Поясните назначение команд СУБД Redis:  incr, decr.
DECR ключ — Уменьшает на единицу значение числа. В случае, если заданный ключ будет содержать строку, будет сгенерирована ошибка.
INCR ключ — Увеличивает на единицу значение числа. В случае, если заданный ключ содержит строку, будет вызвана ошибка.

17. Поясните назначение команд СУБД Redis:  mset, mget.
MSET ключ значение [ключ значение …] — Устанавливает значение ключа / значения ключей, которые переданы в параметрах.
MGET ключ [ключ …] — Возвращает значение ключа / ключей, переданных в параметрах.
18. Поясните назначение команд СУБД Redis:  hset, hget. 
HSET ключ поле значение — Добавляет в хэш поле и значение. Если такого ключа не существовало, он будет добавлен. В случае, если такое поле в хэше уже существует, оно будет перезаписано. 
HGET ключ поле значение — Возвращает значение, которое ассоциировано с полем в хэше  
19. Поясните назначение команд СУБД Redis:  hmset,hmget.    
HMSET ключ поле значение [поле значение …] — Записывает значения в поля хэша.
HMGET ключ поле [поле …] — Получает значение поля / полей указанного хэша.    

20. Поясните назначение команды СУБД Redis: exists.
EXISTS – уведомляет, если ключ существует.

Публикация-подписка или сокращенно pub / sub - это шаблон обмена сообщениями, при котором несколько издателей отправляют сообщения нескольким подписчикам, прослушивая сообщения по определенным каналам через серверное программное обеспечение. Это один из наиболее распространенных шаблонов очереди сообщений. Для этой цели было создано множество систем, среди которых Redis, RabbitMQ, Azure MessageBus и другие...

Redis, высокопроизводительное хранилище структур данных, добавило поддержку pub / sub в дополнение к существующей функциональности. Redis pub / sub построен на трех командах: PUBLISH, SUBSCRIBEи UNSUBSCRIBE.
